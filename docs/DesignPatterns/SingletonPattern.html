Singleton Pattern

此篇原文：Singleton Pattern，詳細的介紹可以讀這篇文章，會有一個步驟一步驟的過程，並且搭配圖片做說明

前言​

Singleton Pattern 是指，應用程式在同一時間，只會同時存在一個實例 (instance)，當所有人需要使用相關功能時，都是使用同一個實例去處理。

此 Pattern 可以解決

  1. 因為同一時間只會有一個實例，如果此實例會在各個地方被使用的話，對比在使用時都實例化，可以有效減少記憶體空間。
  2. 在不同的 function 或是 component 之間，如果使用了不同的實例，可能需要針對資料的順序、儲存做額外的判斷與調整，使用此 Pattern 可以減少來源的判斷，讓程式碼維護性提高。
Counter Class
let instance;
let counter = 0;

class Counter {
  constructor() {
    if (instance) {
      throw new Error("You can only create one instance!");
    }
    instance = this;
  }

  getInstance() {
    return this;
  }

  getCount() {
    return counter;
  }

  increment() {
    return ++counter;
  }

  decrement() {
    return --counter;
  }
}

const counter1 = new Counter();
const counter2 = new Counter();
// Error: You can only create one instance!

我們建立了一個簡單的 Singleton Class，接下來可以使用 Object.freeze 去將 class 實例化出來的物件 凍結 起來並 export 出去

Export the instance
const singletonCounter = Object.freeze(new Counter());
export default singletonCounter;

當然，我們也可以用一般的 object 來完成一樣的需求

Using a regular object
let count = 0;

const counter = {
  increment() {
    return ++count;
  },
  decrement() {
    return --count;
  },
};

Object.freeze(counter);
export {counter};

使用上，就是把實例取出來使用

Demo
import Counter from "./Counter.js";

function Demo() {
  return (
    <button
      onClick={() => {
        Counter.increment();
        console.log("Counter total: ", Counter.getCount());
      }}
    >
      count ++
    </button>
  );
}

應用​

State management in React​

文章最後有說到，像 Redux 或是 React Context 都是有使用 Singleton Pattern ，他們同時符合幾個點

  1. 在 Global 註冊唯一的實例
  2. state 是唯讀的，要修改都要透過特定的 function 去修改
  3. pure function 去改變 state
